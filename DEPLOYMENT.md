# TabNest Documentation Website - Deployment Guide

## Project Overview

This is the official documentation website for **TabNest**, a powerful browser extension for organizing and managing tabs efficiently.

Built with **Docusaurus 3.9** (React-based static site generator), the site includes:
- Comprehensive user documentation
- Developer guides and API references
- Product features showcase
- Blog with release notes and tutorials

## Deployment Information

### GitHub Repository
- **Repository**: https://github.com/binnapro/tabnest-doc
- **Visibility**: Public
- **Deployment**: Automated via GitHub Actions

### Website URL
- **Live Site**: https://binnapro.github.io/tabnest-doc/
- **Build Deployment**: Automatic on push to `main` branch
- **Build System**: GitHub Actions workflow (`.github/workflows/deploy.yml`)

### GitHub Pages Configuration
- **Branch**: `gh-pages` (auto-generated by GitHub Actions)
- **Build Type**: Workflow
- **Source Path**: `/` (root)
- **Custom Domain**: Not configured (using GitHub Pages default domain)

## Maintenance & Updates

### Adding New Documentation
1. Create markdown file in appropriate `docs/` subdirectory
2. File structure automatically creates sidebar navigation
3. Push to main branch
4. GitHub Actions automatically rebuilds and deploys

### Updating Content
1. Edit markdown files
2. Changes are reflected in dev server immediately
3. Push to trigger production deployment

### Customizing Styles
- Modify `src/css/custom.css` for global styles
- Update component inline styles in `src/pages/index.tsx`
- Docusaurus theme colors use CSS variables

## Troubleshooting

### Build Failures
- Check Node.js version (requires >= 20.0)
- Verify all markdown syntax is valid
- Check for broken markdown links

### GitHub Pages Not Updating
- Verify gh-pages branch has latest content
- Check GitHub Actions workflow status
- Clear browser cache
- Wait 1-5 minutes for GitHub Pages to rebuild

### Local Dev Server Issues
- Clear `.docusaurus` cache: `rm -rf .docusaurus`
- Reinstall dependencies: `rm -rf node_modules && npm install`
- Check port 3000 is not in use
